# ADR-003: Core Domain Definition

**Status:** Accepted
**Date:** 2026-07-16
**Author:** Equipe EventOS AI

## Context

The traditional approach would define "Events" as the core domain. However, this limits the platform to pre/post event lifecycle.

## Decision

The **Core Domain** is defined as **Relationship Lifecycle Management**, not Event Management.

The event is just one touchpoint in a continuous relationship:
- Pre-event (marketing, sales, registration)
- During-event (accreditation, access, networking, engagement)
- Post-event (CRM, marketplace, community, re-engagement)

## Consequences

- CRM becomes a first-class citizen, not an add-on
- Platform value extends beyond event dates
- Enables recurring revenue models
- Differentiates from competitors who focus only on event day operations
